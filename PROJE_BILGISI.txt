DataForest - Proje Bilgisi
Tarih: 2026-04-14

1) Proje Özeti
- Proje adı: DataForest
- Amaç: Yapay zeka destekli çevresel risk analizi ve eğilim (trend) tahmini konseptini modern bir web arayüzüyle sunmak.
- Arayüz: Tek sayfa (landing) akışı + scroll ile çalışan canvas “frame sequence” hero.
- Teknoloji: HTML + CSS + Vanilla JavaScript + Canvas API
- Frame sequence: ./images/kolonya/ezgif-frame-001.jpg ... ezgif-frame-150.jpg

2) Klasör ve Dosya Yapısı
- index.html
  -> Sayfanın iskeleti (hero, bölümler, modal, footer).
- style.css
  -> Tüm stil sistemi (navbar, hero, grid/card tasarımları, responsive kurallar).
- app.js
  -> Etkileşimler (navbar scroll efekti, anchor smooth scroll, reveal animasyonları,
     canvas sequence preload + scroll->frame, modal aç/kapat).
- images/
  -> Görseller (hero frame’leri ve içerik görselleri).

3) Çalıştırma (Local)
Bu proje statik dosyalardan oluşur. En sorunsuz yöntem, bir statik server ile açmaktır.

- Node varsa:
  - npx http-server -p 5173
  - ardından tarayıcıda: http://127.0.0.1:5173

Not: Dosyayı doğrudan “file://” ile açmak bazı tarayıcılarda görsel yükleme/önbellek davranışlarını farklılaştırabilir.

4) Önemli Notlar
- Canvas hero “scroll-driven” çalışır:
  - Section yüksekliği (örn. 500vh) scroll mesafesini belirler.
  - Scroll progress 0..1 aralığına map edilir, frame index buna göre seçilir.
- Mobil davranış:
  - Canvas alanının ekrana sığması ve sticky’nin bozulmaması için overflow/viewport ayarları kritik.
  - Sticky bazı mobil tarayıcılarda parent overflow/clip ile bozulabildiği için bu projede buna göre ayarlandı.

5) Dosya Görev Haritası (Nerede Ne Var?)
- index.html
  -> `.hero-sequence-wrapper` içinde `#hero-canvas` ve hero overlay metin blokları.
  -> İçerik bölümleri: proje amacı, metodoloji, grafik kartları, model sonuçları, footer.
  -> Modal: `#image-modal`

- style.css
  -> `.hero-sequence-wrapper`: scroll mesafesi (vh) ve taşma kontrolü
  -> `.sticky-container`: 100vh/100svh/100dvh ile full-viewport “video alanı”
  -> `#hero-canvas`: canvas’ın container’ı kaplaması (absolute inset:0)
  -> `.hero-overlays` ve `.hero-block`: hero üst metin katmanları

- app.js
  -> Frame preload + loader ilerleme göstergesi
  -> `getScrollProgress()` ile scroll progress hesabı
  -> `renderFrame()` ile canvas çizimi (cover fit)
  -> `updateOverlays()` ile metinlerin scroll’a bağlı görünmesi
  -> Modal aç/kapat ve reveal animasyonları
